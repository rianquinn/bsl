Detailed Description

template<typename T>
class bsl::nodelete< T >

Does nothing. As a result, the bsl::unique_owner and friends should optimize away this class when release mode is enabled, thus ensuring that the bsl::unique_owner has the same overhead as a gsl::owner, but is capable of enforcing uniqueness (optimizations only tested with GCC). Once C++20 comes out, the bsl::unique_owner should also be usable in a constexpr as well.

#include <bsl/unique_owner.hpp>
std::int32_t
main() noexcept
{
bsl::unique_owner<std::int32_t> const owns_int{1};
static_cast<void>(owns_int);
}
Note
There is no need to create this class directly as this will be created for you by default as seen in the examples.
Template Parameters
Tthe type provided to bsl::unique_owner. Note that this type information is not used by this class, but would be used by more complicated deleters, which is why this class must also take a template parameter.
